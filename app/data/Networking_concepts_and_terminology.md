## Networking concepts and terminology

**Overview**

This topic presents terminology used when referring to Model 204 intersystem communication. Because the meaning of networking terms could be different in a Model 204 context, Technical Support recommends that all connectivity system administrators read this.

**Basic networking terms and concepts**

In this topic, a computer network is defined as a collection of nodes, connected by software, and so on. When the term "node" is used in this topic, however, it refers to an application node: the point of connection to the network for some application programs.

*   Some examples of communication interfaces
    *   Virtual Telecommunication Access Method (VTAM)
    *   Transmission Control Protocol/Internet Protocol (TCP/IP)

The following figure shows a hypothetical network that consists of three nodes (nodes A, B, and C) that request information from the server node, where the Model 204 SQL process runs.

**Figure description**

*   Essential elements shown in the previous figure:
    *   Exchange of data between application programs at different nodes is referred to as a conversation.
    *   Two programs that use Model 204 are conversation partners.
    *   Client nodes and the workstations are connected to a Model 204 terminal thread.
    *   Each end is defined as an end user.
    *   The TCP/IP (Horizon) network operation is transparent.

**TCP/IP concepts and terminology**

Transmission Control Protocol/Internet Protocol (TCP/IP) is designed for building network interconnections.

*   **TCP**
    *   The transmission control protocol (TCP) controls the transfer of the data.
    *   The TCP layer provides logical connections between processes.
    *   TCP provides reliability in transmission and controls the data flow.
*   **IP**
    *   The internet protocol (IP) provides the routing mechanism for the data.
    *   The internet protocol creates a virtual network view and acts as a layer to hide the underlying physical network.
    *   The IP address is a 32-bit address, usually represented in dotted decimal form.
    *   Network address represents the physical network.
    *   The internet protocol uses an IP address to specify two logical addresses:
        *   IP address specifies an individual host within the physical network.
*   **Ports**
    *   A port is an identifier used by TCP/IP to distinguish one client node from another in a network connecting many client nodes to a single host (or server).

**Identifying Model 204 to TCP/IP**

In setting up a client node for the Model 204 TCP/IP (Horizon) network, you provide an IP address and port number combination to identify this. This example shows valid IP address/port number combinations:

```
192.35.45.128:1063
192.35.45.128 is the dotted decimal notation address
1003 is the port number
```

**Connections**

A TCP/IP connection transports requests and responses between the client nodes and the server node.

**SQL Server overview**

Client/server architecture allows the Model 204 to act as an SQL Server on the host machine, to service client TCP/IP. The SQL Server invokes Model 204 database management system (DBMS) operations, and provides a combination of Model 204 and SQL functionality.

*   **SQL Server**
    *   Compiles SQL syntax strings, checks SQL semantics, optimizes database access, generates code to accomplish SQL requests, and executes the generated code.
    *   Each SQL statement generates data, information, or both for the requesting application.
    *   Serves as an application layer to the SQL Engine, accepting client request packets, processing each packet's requests for the SQL Engine, and returning the SQL processing result to the client.

**SQL Server Front End (SSFE)**

SQL Server provides full SQL processing in one of these locations:

*   Model 204 Address space (z/OS)
*   Virtual machine (VM)
*   Partition (VSE)

**SQL Server associated software**

The SQL Server works in conjunction with the following Model 204 software:

*   SQL catalog
*   SQL communications interface
*   SQL Client Front End (SCFE)

**SQL Server supporting tools**

Table Specification facility (TSF)
Catalog Reporting facility (CCACATREPT)

Transport of the data between the SQL Server and its clients relies on the communication layer: Model 204 TCP/IP (Horizon). These interfaces, which define client to SQL Server connections, are described in this topic and shown in Model 204 SQL processing transport methods.
